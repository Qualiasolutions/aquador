## v1.1 Requirements

Requirements for security audit remediation. Each maps to roadmap phases.

### Security (SEC)

- [x] **SEC-01**: All Supabase tables have RLS enabled with appropriate read/write policies — 9 tables, 24 policies created (migration pending deploy)
- [x] **SEC-02**: Sentry configs set sendDefaultPii: false — no customer PII sent to third party (GDPR)
- [x] **SEC-03**: Admin product search escapes user input before ilike() query
- [x] **SEC-04**: Admin login validates redirect param against allowlist — no open redirect
- [x] **SEC-05**: Permissions-Policy header locks down browser APIs (camera, microphone, geolocation)
- [x] **SEC-06**: CSP tightened — unsafe-eval removed, unsafe-inline retained (Next.js requirement)
- [x] **SEC-07**: CSP media-src restricted from wildcard https: to specific domains

### Testing (TEST)

- [x] **TEST-01**: Stripe webhook route has 21 test cases covering happy path, duplicate events, malformed payloads, and failure scenarios
- [x] **TEST-02**: API route test coverage expanded from 2/14 to 6/14 (checkout, blog, health, heartbeat) — AI assistant skipped due to Jest fetch mocking limitation
- [x] **TEST-03**: CartIcon test fixed — semantic queries replacing incorrect data-testid mocks

### Architecture (ARCH)

- [x] **ARCH-01**: Product type system unified — LegacyProduct deprecated, shop category pages use Supabase Product type
- [x] **ARCH-02**: AI catalogue generated from Supabase at build time via prebuild hook
- [x] **ARCH-03**: Real React error boundary replaces AbortError suppressor — component crashes show branded fallback UI with Sentry logging
- [x] **ARCH-04**: Cart localStorage loading validates item shape with Zod — malformed data cleared and logged
- [x] **ARCH-05**: Cart hydration race condition fixed — useReducer initializer pattern for consistent state
- [x] **ARCH-06**: API error handling consistent — all 12 routes return structured { error: string } responses

### Performance (PERF)

- [x] **PERF-01**: Database indexes added for products (3), blog_posts (3), orders (2) — migration pending deploy
- [x] **PERF-02**: Blog functions use public.ts client — enables static/ISR rendering with 60s revalidation
- [x] **PERF-03**: getRelatedProducts() requires category param — N+1 eliminated (2 queries → 1)
- [x] **PERF-04**: Blog ISR with 60s revalidation + admin dashboard batched queries (10 → 5)
- [x] **PERF-05**: Admin dashboard queries consolidated from 10 to 5 via in-memory aggregation
- [x] **PERF-06**: Product and blog queries use explicit column selection via constants, no select('*')

### Production (PROD)

- [x] **PROD-01**: Sentry tracesSampleRate reduced to 0.1 in production (100% in dev)
- [x] **PROD-02**: Preconnect hints added for Supabase and Stripe domains in layout.tsx
- [x] **PROD-03**: Form accessibility — htmlFor/id pairs, aria-labels on icon buttons, fieldset/legend for radio groups

### Cleanup (CLEAN)

- [x] **CLEAN-01**: Three.js removed from dependencies (~600KB savings), replaced with CSS gradient
- [x] **CLEAN-02**: Legacy Stripe export dead code removed from src/lib/stripe.ts
- [x] **CLEAN-03**: Category system documented as three-part architecture (static file + DB table + DB enum)
- [x] **CLEAN-04**: Magic numbers extracted to named constants (CART_DEBOUNCE_MS, MIN/MAX_QUANTITY, MAX_CART_ITEMS)
