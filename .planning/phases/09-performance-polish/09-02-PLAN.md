**Consolidation strategy:**

1. **Product stats → 2 queries instead of 5:**
   - Query 1: Get all products with select('in_stock, category') - single query for total/in_stock/out_of_stock counts AND category aggregation
   - Query 2: Recent products unchanged

2. **Order stats → 1 query instead of 3:**

3. **Customer count → unchanged (1 query)**

Wait, that's still 5. Let me recalculate...

Actually, combine customers + visitors into admin stats query if possible, OR keep as is since they're separate concerns.

**Refined consolidation (5 queries, down from 10):**
- products aggregate (in_stock, category) → extract total/in_stock/out_of_stock/categoryCount
- products recent 5
- orders aggregate (total, created_at) → extract count/revenue/recent orders

Derive stats in JavaScript:
