<task type="auto">
  <name>Task 1: Fix cart hydration race condition</name>
  <files>src/components/cart/CartProvider.tsx</files>
  <action>
Add mounted state to prevent hydration mismatch between server render (empty cart) and client render (localStorage cart).

**Issue:** Cart Provider hydrates from localStorage in useEffect, but React renders server-side with empty cart first. This causes hydration mismatch warnings when localStorage cart has items.

**Solution:** Track mounted state and render consistent initial state until hydration completes.

Implementation:
1. Add mounted state: `const [mounted, setMounted] = useState(false);`
2. Set mounted in useEffect: `useEffect(() => { setMounted(true); }, []);`
3. Update hydration useEffect to only run when mounted:
```typescript
useEffect(() => {
  if (!mounted) return;

  // Existing localStorage hydration logic...
}, [mounted]);
```
4. Render loading state until mounted:
```typescript
if (!mounted) {
  // Return provider with empty cart during SSR
  return (
    <CartContext.Provider value={{ ...contextValue, items: [] }}>
      {children}
    </CartContext.Provider>
  );
}
```

This ensures server render and first client render match (both show empty cart), then hydration happens after mount.

**Alternative approach (simpler):** Move localStorage read to useState initialization with try-catch:
```typescript
const [cart, dispatch] = useReducer(cartReducer, initialCart, (init) => {
  if (typeof window === 'undefined') return init;
  try {
    const stored = localStorage.getItem(CART_STORAGE_KEY);
    if (!stored) return init;
    const parsed = CartSchema.safeParse(JSON.parse(stored));
    return parsed.success ? parsed.data : init;
  } catch {
    return init;
  }
});
```

Use whichever approach is cleaner and doesn't break existing Zod validation from Phase 8.
