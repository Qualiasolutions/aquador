Create production-ready animated section component that wraps Section.tsx patterns:

**src/components/ui/AnimatedSection.tsx:**

Props interface:
- children: ReactNode
- variant: 'fadeInUp' | 'fadeInDown' | 'fadeInScale' | 'stagger' (default: 'fadeInUp')
- delay?: number (default: 0)
- className?: string
- staggerDelay?: number (for stagger variant, default: 0.1)
- threshold?: number (intersection threshold, default: 0.2)
- disableOnMobile?: boolean (completely disable animations on mobile, default: false)

Implementation:
- Use useScrollAnimation hook from Task 2
- Import variants from scroll-animations.ts (Task 1)
- Wrap children in motion.div with selected variant
- If shouldAnimate is false (reduced motion), render children instantly (no motion.div)
- If disableOnMobile and viewport < 768px, render children instantly
- Match Section.tsx patterns for consistency
- Use cn() for className merging
- Support both 'stagger' variant (for grids) and individual element variants

Mobile optimization:
- Detect mobile via window.innerWidth < 768 or use CSS media query
- On mobile: reduce animation distance by 50%, faster duration (0.4s instead of 0.6s)
- Provide escape hatch via disableOnMobile prop for complex layouts

Example usage in docstring:
```tsx
<AnimatedSection variant="fadeInUp">
  <h2>Animated Heading</h2>
</AnimatedSection>

<AnimatedSection variant="stagger" staggerDelay={0.15}>
  {products.map(product => <ProductCard key={product.id} {...product} />)}
</AnimatedSection>
```

Component must be 'use client' directive (uses hooks and Framer Motion).
