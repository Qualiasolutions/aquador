  <files>
src/lib/animations/micro-interactions.ts
  </files>
  <action>
Create `src/lib/animations/micro-interactions.ts` with comprehensive micro-interaction patterns:

Export `hoverVariants` object with preset hover effects:
- `lift`: scale: 1.02, y: -4, subtle shadow increase (for cards, large buttons)
- `glow`: scale: 1.05, add glow/shadow with gold-500 color (for primary CTAs)
- `scale`: scale: 1.05 only (for small interactive elements)
- `underline`: width expansion for text underlines (for links)
- `tilt`: subtle rotateX/rotateY for 3D depth (for cards, avoiding mobile)
- All use transition duration 0.2s with cubic-bezier(0.22, 1, 0.36, 1) easing

Export `tapVariants` object for click/touch feedback:
- `shrink`: scale: 0.95 (universal tap feedback)
- `shrinkHard`: scale: 0.9 (for buttons needing strong feedback)
- `pulse`: scale: [1, 1.1, 1] with 0.3s duration (for success states)

Export `focusVariants` for keyboard navigation:
- `ring`: outline with gold color, 2px offset (WCAG 2.1 AA compliant)
- `scale`: slight scale increase + ring (for interactive elements)

Export `loadingVariants` for async operations:
- `spin`: continuous rotation (for spinners)
- `pulse`: opacity oscillation (for loading text)
- `dots`: staggered dot animation (for "..." loading indicators)

Export utility functions:
- `createHoverEffect(type, options)`: Factory for custom hover animations
- `combineVariants(variant1, variant2)`: Merge multiple animation objects
- `withReducedMotion(variants, fallback)`: Wrapper that returns fallback if reduced motion

Follow Phase 12 patterns:
- Use EXPO_EASE constant from scroll-animations.ts
- Document performance characteristics (GPU vs CPU animations)
- Include reduced motion alternatives for each variant
- Add TypeScript interfaces for all variant types
- Extensive JSDoc with usage examples

Special considerations:
- Touch devices: Use `whileTap` instead of `whileHover` for mobile
- Avoid `transform: rotate` on mobile for performance (use scale only)
- Keep animations under 300ms for perceived instant feedback
- Use `will-change: transform` hints for GPU acceleration
