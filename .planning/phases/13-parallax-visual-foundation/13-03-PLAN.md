Create `src/lib/performance/animation-budget.ts` performance monitoring system:

Export `useAnimationBudget()` custom hook:
- Track FPS using requestAnimationFrame loop
- Calculate average FPS over 1-second windows
- Detect frame drops (< 55fps threshold)
- Return object: `{ averageFps, isPerformant, shouldSimplify }`
- If `shouldSimplify` is true (sustained < 55fps), recommend disabling complex effects

Export `trackAnimationPerformance()` utility function:
- Accept `animationName` and `callback` parameters
- Measure callback execution time using `performance.now()`
- Log to console in development: `[Animation] {animationName}: {duration}ms`
- Track in Sentry in production if duration > 16ms (dropped frame threshold)

Export `AnimationBudgetProvider` context provider:
- Wraps app to provide global performance state
- Automatically simplifies animations if sustained poor performance detected
- Provides `shouldUseSimplifiedAnimations` boolean to all consumers

Include TypeScript interfaces:
```typescript
interface AnimationBudget {
  averageFps: number;
  isPerformant: boolean;
  shouldSimplify: boolean;
}

interface AnimationBudgetContextValue {
  budget: AnimationBudget;
  shouldUseSimplifiedAnimations: boolean;
}
```

Performance thresholds:
- Excellent: 60fps sustained (full effects)
- Good: 55-60fps (full effects)
- Acceptable: 45-55fps (simplified effects)
- Poor: < 45fps (minimal effects, static transforms only)

Add JSDoc with performance optimization guidance.
