Create `src/hooks/useSwipeGesture.ts`:
- Custom hook for detecting horizontal swipe gestures on mobile
- Options: `{ threshold?: number, onSwipeLeft?: () => void, onSwipeRight?: () => void, enabled?: boolean }`
- Default threshold: 50px (minimum distance for swipe to trigger)
- Track touch start X/Y coordinates and compute delta on touch end
- Only trigger horizontal swipe if deltaX > deltaY (prevent conflict with vertical scroll)
- Include velocity detection: swipe must complete within 300ms to count (prevents slow drags)
- Return: `{ ref, isSwiping, swipeDirection }` where ref is a React ref to attach to container
- Handle edge cases: multi-touch (ignore), touch cancel, passive event listeners
- Export `SwipeDirection` type: 'left' | 'right' | null
- 'use client' required
- TypeScript interfaces for all options and return types

Create `src/components/shop/SwipeableProductGrid.tsx`:
- Wraps product grid content with swipe gesture detection
- Props: `categories: Category[]`, `currentCategorySlug: string`, `children: ReactNode`
- On swipe left: navigate to next category (using Next.js `router.push`)
- On swipe right: navigate to previous category
- Category order derived from `categories` array (from categories.ts)
- Show subtle edge indicator during swipe (thin gold line at screen edge, 2px)
- Desktop: swipe disabled (enabled prop = false for useSwipeGesture)
- Visual feedback: during active swipe, slight x-translation follows finger (max 40px)
- Import `useRouter` from next/navigation for programmatic navigation
- Import `useSwipeGesture` hook
- At top/bottom of grid, show tiny category name hint: "< Women's" on left, "Niche >" on right
